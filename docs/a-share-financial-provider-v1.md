# A 股财务 Provider V1

## 结论

A 股财务 Provider V1 已在当前 56 只 A 股股票池完成真实全量运行：56/56 有状态记录，56/56 最新报告满足 V1 核心字段门槛，0 家抓取失败。数据由新浪财经公开 `CompanyFinanceService.getFinanceReport2022` 接口获取，底层金额统一为人民币元，比例统一使用小数（`0.15` 表示 `15%`），缺失值始终为 `null`。

本版本暂不接入默认 `data:refresh`。原因不是覆盖失败，而是 Provider 仍依赖无 SLA 的第三方公开接口，且需要更多跨日运行记录验证稳定性；当前保留独立命令，避免一次外部接口波动影响行情等既有刷新任务。

接口实测日期：2026-07-10。实测对象包括 600519（沪市普通公司）、300308（深市科技公司）、688017（科创板公司）和 000001（银行，当前池外）。三张报表均返回最近一期和历史年度报告；截至实测时最近一期为 2026-03-31，并包含 2025 年报及 2025 年三个季度报告。当前股票池没有北交所公司，因此未产生北交所正式覆盖结果；代码适配器支持 `BJ` 前缀并有单元测试。

接口元数据表明报表来源为“定期报告”，可区分合并/母公司口径、审计状态、披露日期和源更新时间。接口无 Token、无公开 SLA；未找到可保证商业再分发的专门许可声明，本项目仅将其用于内部研究数据生成，采用串行请求、每只股票四个端点、0.12 秒请求间隔、15 秒超时和最多 2 次有限重试，不绕过登录、验证码或反爬限制。

## 架构与文件

- `scripts/a_share_financials/provider.py`：代码适配、HTTP 客户端、缓存、超时与有限重试。
- `scripts/a_share_financials/core.py`：字段映射、单位归一、版本选择、单季度推导、同比/环比和数据校验。
- `scripts/fetch-a-share-financials.py`：全量/单股/报告期/缺失刷新/dry-run 命令入口。
- `scripts/validate-a-share-financials.py`：generated 数据专项门禁。
- `src/data/real/a-share-financials.generated.json`：稳定排序的 V1 生成文件。
- `src/types/marketData.ts`：统一 TypeScript 财务模型。
- `src/components/stock/StockDetailDrawer.tsx`：个股详情页有限接入。

缓存位于 `data-cache/a-share-financials-v1/`，已由现有 `.gitignore` 排除。缓存与正式 generated 文件分离；失败时如存在上一版真实数据，会保留旧报告并标记 `stale`、记录 `lastSuccessfulFetchAt` 和 `currentFetchError`，不会用示例数据替代。

## 命令

```bash
npm run data:fetch:financials:a
npm run data:validate:financials:a
npm run test:financials:a
python scripts/fetch-a-share-financials.py --stock 300308 --dry-run
python scripts/fetch-a-share-financials.py --period 2026-03-31 --missing-only
```

结构性错误使用非零退出码；单家公司网络失败会形成明确错误或 stale 记录，不会导致其他公司从结果中消失。`--no-cache` 可强制重新请求，默认缓存用于避免重复抓取。

## 标准模型与口径

类型层级包括 `AShareFinancialData`、`FinancialReport`、`IncomeStatementMetrics`、`BalanceSheetMetrics`、`CashFlowMetrics`、`FinancialDerivedMetrics`、`FinancialDataProvenance` 和 `FinancialFetchStatus`。公司状态支持：

- `success`
- `partial`
- `not_applicable`
- `source_unavailable`
- `fetch_error`
- `validation_error`
- `stale`

每份报告保存股票、市场、公司、报告期、报告类型、财年/季度、公告日期、合并/母公司口径、币种、源单位、标准单位、Provider、源标识、抓取/源更新/生成时间、重述与推导信息、字段覆盖率和字段状态。

金额统一为人民币元。新浪三表和关键指标接口返回值已经是元，不按数值大小猜单位，`sourceUnit=元`、`normalizedUnit=元`、`normalizationFactor=1`。正式报告抽样中工业富联使用千元披露，核验时乘以 1,000 后与 Provider 的元口径比较。

## 累计值、单季度与增长率

利润表和现金流量表同时保留 `cumulative` 与 `singleQuarter`：

- Q1：Q1 累计值即单季度值，不标记为差分推导。
- Q2：H1 累计 - Q1 累计。
- Q3：Q3 累计 - H1 累计。
- Q4：FY 累计 - Q3 累计。

只有期间型字段参与相减。资产、负债、权益、应收、存货、借款和商誉等时点字段仅保存在 `balanceSheet`，不会进入 `singleQuarter`。缺少兼容前序报告或重述状态不一致时不推导，结果为 `null` 并记录原因。

单季度同比比较上年同季度的单季度值；环比比较相邻单季度，Q1 可与上年 Q4 比较。分母缺失时比例为 `null`；分母为 0 时保留 `changeAmount`，比例为 `null` 且原因为 `denominator_zero`；负基数使用绝对基数计算变化比例并记录 `baseSign=negative`，避免把数学结果误读为常规增长。

## 金融行业处理

银行、保险、券商等企业不强制计算制造业毛利率、应收收入比、存货收入比和研发费用率，不适用字段标记为 `not_applicable`。通用字段仍保留。V1 不生成净息差、新业务价值或经纪业务收入等行业专属指标。

当前 56 只 A 股池中金融行业公司为 0，因此实际生成统计为 0；000001 平安银行的接口返回结构已实测，金融分支由专项单元测试覆盖。没有把池外银行加入 generated 数据，也没有扩充股票池。

## 前端规则

个股详情页优先展示最新报告期、单季度营业收入/归母/扣非、同比/环比、毛利率、净利率、经营现金流、应收、存货、研发费用率、公告日、来源、抓取时间、状态和推导说明。

- 缺失：`暂未获取`
- 不适用：`不适用`
- 抓取失败：`数据获取失败`
- stale：`数据已过期`
- 差分单季度：`由累计值推导`

页面继续把财务 `quality` 纳入 `DataQualityBadge` 风险优先级，`partial/stale/error` 不会被来源标签掩盖。

## 已知限制与后续

1. 新浪公开接口没有可用性 SLA；应继续记录跨日刷新成功率和字段漂移。
2. 当前池无金融公司，合并前可以接受，但金融行业真实池内覆盖仍需未来股票池自然包含后再验收。
3. `isRestated` 只能在源端明确提供版本信息时可靠判断；无法判断的重复版本标记 `conflicted`。
4. 生成文件约 4.84 MB，构建可通过但会增加初始包体；后续优先按公司拆分并懒加载，不在 V1 重构页面。
5. 未实现港股财务、公告解析、研报、机构一致预期、估值模型和行业专属财务指标。
6. 未接入默认 `data:refresh`；待至少多次跨日运行、接口字段漂移监控和包体拆分后再评估。
